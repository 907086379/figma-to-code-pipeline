# 新旧设计稿对比：怎样更利于自动还原成代码（设计师读本）

写给**产品 / 视觉 / 交互**：对比「旧参考稿」和「新 Flowbite 三屏」在**同一套协作方式**下，哪边更省事、你们还可以顺手多做哪几步。

---

## 1. 在对比什么

- **新稿**：登录 1 → 登录 2 → 设置 / 首页，三屏分开交付；画板统一、信息层级清楚。
- **旧侧参照**：历史上更「密」的通话控件条一类页面（小按钮多、状态文案多），对**状态写全、命名清楚**要求更高；对照用，不表示旧稿一定差。

---

## 2. 新稿做得好的地方（建议保持）

### 2.1 图层名字像「页面结构」

常见名字如：`Login`、`Form`、`Card`、`Inputs`、`Navbar`、`Settings` 等。接手的人一眼能分出卡片、表单、顶栏，不用在无名 Frame 里猜主内容区。

### 2.2 颜色、间距、字阶尽量走变量

和 Flowbite / 变量库一致时，后面还原成界面会省很多「逐个对 hex」的时间。

### 2.3 一屏一链接，职责清楚

三屏各给独立链接，每屏做什么一眼能辨。**从 A 到 B 的跳转**，请用 **Annotations** 或工单写一句，不要只靠画布上的原型连线（原因见 **3.5**）。

### 2.4 同一套顶栏多屏复用

例如顶栏在多屏里名字一致，后面做公共顶栏更顺。

### 2.5 和《高还原协作规范》更贴

结构清楚、变量多、命名有语义，整体更符合「能还原、能对照」的习惯。

### 2.6 一直在用 Annotations（请继续保持）

用 Annotations 写清「这里容易误会」的点（占位还是真填、按钮在流程里干什么等），**会跟着设计一起交给后面环节**，不画进最终界面像素里。

画布上的 **原型连线 / 流程连接器** 给人看很直观，但**自动从 Figma 拉出去给工具用时，通常带不上「连到哪一帧」这句话**；所以**页面怎么转、逻辑怎么走**，更要靠 Annotations 里的 **from → to**（见 **3.5**）。

### 2.7 增量开发：大页面里的小组块可另给链接（按需）

**整页**照旧保留一条「全屏 / 主 Frame」的设计链接即可。若本轮是**增量**：只改大页面里的某一小块（例如弹层、侧栏、富工具条、复杂卡片），建议在工单或交接里**额外**附上**只选中该组块**后从浏览器复制的 **带 `node-id` 的链接**（与整页链接**并存**，不替代整页）。

这样接手方对稿范围小、迭代快。**不必**把每个小组件都常态化拆链接——**有增量、有独立验收需求时再加**即可。

---

## 3. 还可以再顺手做一点的地方

下面这些不是挑刺：稿面有一点歧义，后面就要多轮问你们——标一句就省很多事。

### 3.1 Logo：拆字拼色（像渐变）

字母拆开、每字不同色，观感可以很像渐变，**完全合理**。

只是：**自动导出的文案清单**里容易出现 `TEXTR`、单字母等很多条，**翻译 / 校对文案时容易吵**；变量表里也可能不好用「一条渐变」概括。

**省事做法（任选）**：封面或工单写一句——「顶栏 Logo 是拆字渐变字标，**不参与**多语言 / 文案验收」。

**若想稿里更标准**，可在 Figma 里改用下面任一方式（和研发对齐用词即可）：

| 目标 | 在 Figma 里怎么做 |
| --- | --- |
| 整段字一条平滑渐变 | 文本填充里用 **线性 / 径向渐变** |
| 渐变只出现在字形里 | **蒙版**：下面放渐变，上面文字 → **使用蒙版** |
| 字标当整块图形 | 字轮廓化或 **联集** 成一块矢量，再整体填渐变 / 导出图 |

### 3.2 输入框：像「已填写」还是「占位」

若默认态看起来像已经填了邮箱，后面容易按「默认就有字」去做，和「只是占位提示」不一致。

**建议**：默认态用 **placeholder**；若要展示填完后的样子，用 **Variant** 或单独一帧标「已填写」。若已在 **Annotations** 里写清「预填 vs 占位」，更好。

### 3.3 设置 / 工作台被当成「表单页」猜

结构里输入区多的时候，自动说明有时会按「表单」来写，和真实「设置页」不完全一致。

**建议**：在 **Annotations** 或封面写一句：这是工作台 / 设置，主导航、抽屉、保存大致怎样。特别复杂的可以拆成「只交侧边栏」「只交内容区」两个链接分步做。

### 3.4 图标特别多、名字很技术

能进库尽量用 **Instance**。交付时一句话说清：图标能不能继续用稿里的链接、还是要换成你们本地的图。

### 3.5 跨屏跳转：不要只依赖画布上的线

**原型连线、connector** 给团队评审很好用。

但当前从 Figma **自动拉给协作工具**时，**一般不会带上「点这里会到哪一帧」这条信息**——接手的人**看不到**和你们画布上那条线等价的「目标页说明」。

**请养成习惯**：

- **逻辑 / 页面怎么转**：在相关画板或组件上写 **Annotations**，一句话 **from → to**（写清**目标画板名字**即可；需要精确到节点时，**直接粘贴浏览器地址栏里的设计链接**，或复制链接里的 `node-id=…`，**保持你们习惯的写法**，不必手改格式——工程侧可用脚本批量统一）。
- 原型线**继续画**；Annotations 当作**给后面环节的补充说明**，避免只能靠猜。
- 工单或封面重复写一句跳转也可以。

**from → to 示例（直接沿用链接里的 `node-id` 写法即可）：**

- `11038-684`（登录邮箱步）→ 点 **Next** → `11101-839`（登录密码步）  
- `11101-839`（登录密码步）→ 点 **Sign in / 主操作** → `11147-1506`（设置 / 首页）

整段 **Figma 设计链接** 贴在 Annotations 里也可以，同样有效。

---

## 4. 一句话对照旧侧

旧参照那种「又密又多状态」的条：**状态写全、跳转写清**更费神。你们新稿在**结构、变量、分屏**上已经轻松一截；接下来最划算的是：**Logo 交付说明一句、占位 / 已填分清楚、流转在 Annotations 里写 from → to（别只靠连线）**；**大页增量只改小组件时，可为该块另附一条设计链接（见 2.7）**。

## 5. 自检清单（提交前可打勾）

- [ ] 关键页面是否**各有一条独立设计链接**（或写清范围）？
- [ ] **增量只改小组件时**：是否已为该组块补充**一条独立设计链接**（选中组块后复制链接即可；与整页链接并存，见 **2.7**）？
- [ ] Logo 拆字渐变是否在工单 / 封面标明「不参与文案 / 多语言」？若要进变量表，是否已改成整段渐变 / 蒙版 / 单块矢量之一？
- [ ] 输入默认态是否 **placeholder**，「已填」是否单独 Variant？
- [ ] 颜色、字阶、间距是否主要走 **变量**？
- [ ] 易误会处是否用 **Annotations** 写过短句（**已这样做的请保持**）？
- [ ] 非表单页是否在 **Annotations**（或原型说明 / 封面）里写了导航、保存、弹层等一句？
- [ ] **跨屏 / 流程**：是否在 **Annotations**（或工单）里写了 **from → to**？**不要只靠**画布原型线——**自动拉稿时通常读不到线的目标页**。
- [ ] 图标 / 插画是否有一句「能不能用稿里资源、要不要换本地」的说明？
