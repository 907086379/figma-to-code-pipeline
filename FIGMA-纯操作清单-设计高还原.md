# Figma 纯操作清单（设计师高还原版）

> 目标：不依赖代码修改，仅在 Figma（含插件/MCP）内完成可高还原交付。

## A. 仅用 Figma 原生操作（必做）

## 1) 结构与布局

- [ ] 所有页面主区域使用 Auto Layout（避免纯绝对定位堆叠）。
- [ ] 每个模块分为：容器 / 内容 / 操作区三层，不混在同一层级。
- [ ] 关键容器配置 Constraints（拉伸、固定、居中）明确。
- [ ] 同类模块层级一致（不要同一类卡片每页结构都不一样）。

## 2) 命名与组件

- [ ] 页面、Frame、组件、关键分组都有业务语义名（禁用 `Frame 1` / `Group 2`）。
- [ ] 可复用块全部组件化（Create component）。
- [ ] 状态差异使用 Variants，不用复制多份散件。
- [ ] 组件名统一 PascalCase（如 `UserCard`, `FilterPanel`）。

## 3) Token（Variables/Styles）

- [ ] 颜色全部绑定变量，不直接散落 Hex。
- [ ] 字体（字号/行高/字重）走统一文本样式命名。
- [ ] 间距/圆角/描边有统一变量或固定规范值。
- [ ] 深浅色主题（如有）放在变量模式中，不手动逐层改色。

## 4) 文案与状态

- [ ] 所有用户可见文案已定稿（不留“文字文字文字”占位）。
- [ ] 表单类至少给：默认、聚焦、错误、禁用。
- [ ] 选择类至少给：selected / unselected。
- [ ] 展开类至少给：collapsed / expanded。
- [ ] 空态、异常态、加载态（若业务存在）至少给 1 个明确画面。

## 5) 交互与流程

- [ ] Prototype 中关键点击路径有连线（from -> to）。
- [ ] 分支路径明确（成功 / 失败 / 无权限 / 空结果等）。
- [ ] 弹层、抽屉、下拉等出现与关闭路径可追踪。

## B. 用插件可提升效率（选做但推荐）

## 6) 推荐插件操作

- [ ] 用命名规范插件批量清理无语义层名。
- [ ] 用样式/变量审计插件排查未绑定 token 的对象。
- [ ] 用重复层/重复样式检查插件清理“几乎一样但不一致”的组件。
- [ ] 用对齐与间距检查插件统一列表、卡片、表单栅格节奏。

> 插件名称可按团队习惯选，不限定具体品牌；目标是“批量发现不一致”。

## C. 用 MCP（Figma -> 代码链路）前的设计自检（推荐）

## 7) 设计师在 Figma 侧可先完成的 MCP 友好动作

- [ ] 每个待交付页面记录 node-id（用于精准定位）。
- [ ] 对关键组件补一份状态覆盖截图（或状态集合页）。
- [ ] 若使用图片资源，标注“装饰图 / 业务内容图 / 图标”用途。
- [ ] 对无法在稿内表达的逻辑，追加注释：触发条件、结果、降级方案。

## 8) 交付前 60 秒终检（必须）

- [ ] 结构可还原：层级清晰 + Auto Layout 完整。
- [ ] 视觉可映射：颜色/字体/间距已 token 化。
- [ ] 状态齐全：不是只有默认态。
- [ ] 交互闭环：关键路径与分支可追踪。
- [ ] 命名可检索：无大量无语义名称。
